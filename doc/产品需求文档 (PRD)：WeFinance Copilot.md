# 产品需求文档 (PRD)：WeFinance Copilot



> 项目名称： WeFinance Copilot (AI 驱动的个人智能财务助理)
>
> 目标赛事： 2025深圳国际金融科技大赛（人工智能赛道）
>
> 文档版本： 1.0
>
> 日期： 2025年11月5日



## 1. 项目概述





### 1.1 项目愿景



WeFinance Copilot 是一个“AI原生”的智能财务助理 1。它不只是一个附加的聊天机器人，而是深度嵌入数字银行APP的核心体验层。我们的愿景是解决当前金融服务中最大的用户体验痛点：复杂性、不透明和缺乏个性化 2。

本项目旨在利用生成式AI (GenAI) 和数据分析技术，为用户提供一个能对话、能预测、能解释的“私人CFO”，实现从“被动应答”到“主动式财务健康管理”的飞跃。



### 1.2 核心问题 (用户痛点)



当前的数字银行体验充满了摩擦点，导致用户焦虑和流失：

- **账单与预算的复杂性：** 用户面对复杂的银行账单和难以坚持的预算工具，感到不堪重负 [User Input]。
- **金融产品的“黑盒”：** 传统的理财推荐充满了用户无法理解的术语，且缺乏对其背后逻辑的透明解释，导致用户信任度低 3。
- **无效的客户服务：** 现有的客服机器人无法理解上下文，导致用户在遇到复杂问题时（如欺诈、复杂交易）体验不佳 2。
- **安全与便捷的冲突：** 用户在寻求便捷服务的同时，也极度担心其敏感金融数据的安全 2。



## 2. 战略定位与竞赛目标



WeFinance Copilot 的设计严格对标2025年大赛人工智能赛道的所有评估标准 [User Query]。

| **大赛评估标准 (摘自 [User Query])**                  | **WeFinance Copilot 核心对策**                               |
| ----------------------------------------------------- | ------------------------------------------------------------ |
| **1. 显著改善用户体验**（更自然的交互、更智能的推荐） | **[F1] 对话式财务问答**：使用自然语言处理（NLP）和GenAI，将复杂的账单和术语“翻译”成“人话”。 |
| **2. 提供个性化服务**（满足不同用户群体的差异化需求） | **[F3] 个性化推荐引擎**：结合用户画像、风险偏好和实时消费数据，生成定制化的投资和预算建议 1。 |
| **3. 支持可解释性输出**（让用户理解推荐或分析的依据） | ** 可解释性 (XAI) 核心**：**本项目的灵魂**。所有AI建议（理财、预算、异常提醒）都将配备“AI为什么这么建议？”按钮，提供清晰、透明的决策依据。 |
| **4. 满足实际应用的可行性**（兼顾实时性与可扩展性）   | **[F4] 主动式异常检测**：通过轻量级模型实时分析交易流，主动预警异常支出或潜在风险，展示“主动式服务” 2。 |
| **5. 与数字银行APP更好地集成**（额外加分项）          | **[架构] AI原生设计**：Copilot 被设计为一个*嵌入式*智能层，而非独立APP。它能主动在APP的各个角落（如账单页、投资页）弹出“洞察卡片”，实现无缝集成 1。 |



## 3. 目标用户画像



1. **金融新手 (P0)**：
   - **特征**： 刚开始工作的学生或职场新人，对理财概念模糊，希望开始储蓄但无从下手。
   - **痛点**： 看不懂金融术语，害怕风险，需要一个“领路人”。
   - **需求**： 用“人话”解释账单，提供简单、安全、可理解的理财入门建议。
2. **繁忙的专业人士 (P1)**：
   - **特征**： 收入稳定，但没有时间或精力管理复杂的财务。
   - **痛点**： 预算工具太耗时，容易忘记账单，希望“一站式”自动管理。
   - **需求**： 自动化的账单分类、智能的预算提醒、主动的异常支出警示。



## 4. 功能需求 (Functional Requirements)





### P0：竞赛Demo核心功能 (MVP)



| **ID** | **功能模块**               | **详细需求描述 (User Story)**                                |
| ------ | -------------------------- | ------------------------------------------------------------ |
| **F1** | **智能账单分析器**         | 1. 用户可上传模拟的银行账单CSV文件。  2. 系统能自动对交易进行智能分类（餐饮、交通、投资等）。  3. AI能自动生成月度/周度消费总结报告，并以“人话”形式高亮关键洞察（如：“您本月在‘外卖’上的支出比上月增加了30%”）。 |
| **F2** | **对话式财务顾问**         | 1. 用户可以使用自然语言进行提问（如：“我这个月还能花多少钱？”、“什么是ETF？”）。  2. AI应能结合[F1]的账单分析结果和上下文，提供个性化回答（如：“您本月预算剩余500元，主要支出在...”）。 |
| **F3** | **可解释的理财建议 (XAI)** | 1. 用户完成一个简单的风险偏好问卷（如：保守型、稳健型、激进型）。  2. 用户输入投资目标（如：“我想存一笔买车的钱”）。  3. AI生成一个简单的、个性化的资产配置建议（如：“建议您70%配置于A基金，30%配置于B理财”）。  4. **[核心]** 建议下方必须有一个**“为什么？”**按钮，点击后AI会用通俗易懂的语言解释推荐逻辑（如：“因为A基金是低风险债券，符合您‘保守型’偏好；B理财... ”）。 |

## 5. 非功能需求 (Non-Functional Requirements)



| **ID**    | **类别**           | **需求描述**                                                 |
| --------- | ------------------ | ------------------------------------------------------------ |
| **NFR-1** | **可解释性 (XAI)** | **(最高优先级)** 所有由AI/ML模型生成的推荐、预测或警示，都必须在技术上可溯源。系统必须能向用户提供一个简化的决策逻辑链。 |
| **NFR-2** | **隐私与数据安全** | 1. **[核心设计]** 严格遵守“数据最小化”原则 3。  2. 方案设计：用户上传的原始账单(CSV)应**优先在本地浏览器/客户端**进行解析和脱敏处理。  3. 只有**聚合后的、匿名的统计特征**（如“餐饮支出占比30%”）才会被发送到后端服务器进行AI分析，绝不传输原始交易详情。 |
| **NFR-3** | **性能 (Demo)**    | 所有交互式查询（问答、账单分析）的响应时间必须在3秒内，以确保现场Demo的流畅性。 |
| **NFR-4** | **可行性**         | 必须使用成熟的开源框架（如Streamlit, FastAPI）构建原型，确保在竞赛周期内可交付、可演示。 |
