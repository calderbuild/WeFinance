# Codex任务3：竞赛演示材料准备 - 详细实施指南

## 任务概述

**目标**：准备完整的竞赛演示材料，包括高质量UI截图、演示视频和竞赛PPT。

**预计时间**：3-4小时

**输出成果**：
- 8张UI截图（1920x1080，PNG格式）
- 1个演示视频（3-5分钟，MP4格式，带字幕）
- 1份竞赛PPT（15-20页，专业设计）

**重要性**：这些材料直接影响评委对项目的第一印象，决定能否进入决赛。

## 任务3.1：UI截图准备（1.5小时）

### 前置准备

**1. 启动应用**
```bash
conda activate wefinance
streamlit run app.py
```

**2. 准备测试数据**

在浏览器中完成以下操作（生成真实数据）：

**步骤1 - 上传账单**：
- 导航到"账单上传"页面
- 上传 `assets/sample_bills/bill_mixed.png`（包含4笔交易）
- 等待OCR识别完成
- 确认交易记录显示正确

**步骤2 - 设置预算**：
- 在侧边栏找到"月度预算"设置
- 输入 `8000`
- 确认预算显示为 ¥8,000

- 导航到"消费分析"页面

## 任务3.2：演示视频录制（2小时）

### 前置准备

**1. 安装录屏工具**

**Windows**：
- OBS Studio（推荐）：https://obsproject.com/
- Windows Game Bar（内置）：Win + G

**Mac**：
- QuickTime Player（内置）
- OBS Studio

**Linux**：
- SimpleScreenRecorder
- OBS Studio

**2. 录制设置**

**OBS Studio配置**：
1. 打开OBS Studio
2. 场景 → 添加 → 显示器捕获
3. 设置 → 输出：
   - 输出模式：简单
   - 录制质量：高质量，文件大小中等
   - 录制格式：MP4
   - 编码器：x264
4. 设置 → 视频：
   - 基础分辨率：1920x1080
   - 输出分辨率：1920x1080
   - 帧率：30 FPS

**音频设置**：
- 麦克风音量：适中（不要太小或太大）
- 降噪：开启（如果有）
- 背景音乐：可选（音量要低于人声）

- 预先练习操作流程2-3遍
- 准备好旁白稿（可以打印出来对着读）

**2. 录制过程**
- 开始前深呼吸，保持语速平稳
- 鼠标移动要流畅（不要抖动）
- 点击要明确（不要多次点击）
- 等待页面加载时不要急躁

**3. 常见问题处理**

**问题1**：说错话了
- 解决：继续录完，后期剪辑时删除

**问题2**：页面卡顿
- 解决：暂停录制，刷新页面，重新开始这一段

**问题3**：AI回答太慢
- 解决：后期加速播放（1.5x-2x速度）

**问题4**：声音太小或太大
- 解决：录制前测试音量，调整麦克风增益

- 使用项目符号（•）或数字（1. 2. 3.）

- 简单的淡入淡出
- 避免花哨的飞入旋转
- 每页动画<3个
- 演示时手动推进（不要自动播放）

## 常见问题

**Q: 截图时页面加载太慢怎么办？**
A: 提前加载好页面，等完全渲染后再截图。或者重启应用。

**Q: 录制视频时说错话了怎么办？**
A: 继续录完，后期剪辑时删除失误片段。不要停下重录。

**Q: PPT设计不够专业怎么办？**
A: 使用PowerPoint或Keynote的内置模板，选择"商务"或"科技"类别。

**Q: 视频文件太大怎么办？**
A: 导出时降低比特率（8-10 Mbps），或使用HandBrake压缩。

**Q: 没有视频剪辑经验怎么办？**
A: 使用最简单的工具（Windows Clipchamp / Mac iMovie），只做基本的剪切和字幕添加。
