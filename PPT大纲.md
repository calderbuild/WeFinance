## 03 解决方案 (2分钟)

### 第7页：核心解决方案 - "三位一体"

**方案架构图**：
```
          用户问题

    │                   │                   │
    └─────────┬─────────┘
```

**解决方案1：Vision LLM一步识别**

**技术路线演进**：
```
传统方案（已废弃）：
图片 → PaddleOCR → 纯文本 → GPT-4o → JSON
问题：PaddleOCR识别率0%（对合成账单）

当前方案：
图片 → GPT-4o Vision → 一步到位 → JSON
优势：识别率100%，代码量减少40%
```

**关键突破**：
- Prompt工程：强制模型"先数有几行，再逐行提取"
- 识别率从30% → 100%（11月15日改进）
- 实测数据：7张测试图片全部正确

**解决方案2：XAI可解释推荐**

**技术架构**：
```
规则引擎（决策）+ LLM（解释）

示例流程：
1. 用户填写风险问卷 → "保守型"
2. 规则引擎过滤产品 → 债券基金 + 货币基金
3. 生成配置方案 → 70%债券 + 30%货币
4. 记录决策日志 → JSON格式
5. LLM生成解释 → 自然语言
```

**用户理解度测试**：
- 5名非金融背景用户
- 平均评分：4.2/5星
- 反馈："第一次看懂为什么推荐这个基金"

**解决方案3：主动异常检测**

**检测逻辑**：
```
异常类型：
├─ 金额异常: amount > mean + k*std (k自适应)
├─ 时间异常: 22:00-6:00 大额消费(>500元)
├─ 频率异常: 同商户单日消费>3次
└─ 重复扣费: 同商户同金额7日内重复
```

**自适应阈值**：
- 样本<10: 不检测（数据不足）
- 样本10-30: k=2.5（宽松，降低误报）
- 样本>30: k=1.5（严格，提高灵敏度）

**实测效果**：
- 召回率：75%（3/4异常被检出）
- 误报率：6.25%（1/16正常消费误判）
- F1分数：0.80（可用水平）

### 第10页：核心技术组件深度剖析

**组件1: Vision OCR Service**

**Prompt工程关键突破**：
```python
# 原始Prompt（识别率30%）
"请提取图片中的交易记录..."

# 优化后Prompt（识别率100%）
"""
重要：
1. 首先统计图片中有多少笔交易（有几行独立金额就有几笔）
2. 然后逐行提取每一笔的详细信息
3. 确保 transactions 数组长度 = transaction_count
"""
```

**改进效果**：
- 多行识别：从"只看第一行"到"逐行提取"
- 测试结果：10张图片，3张正确 → 10张全对
- 关键洞察："引导模型思考结构，比优化解析代码有效"

**组件2: Chat Manager (RAG)**

**上下文注入流程**：
```
用户查询："我这个月还能花多少？"
    ↓
从 session_state 提取数据
    ├─ transactions: 40笔消费记录
    ├─ budget_config: 月预算5000元
    └─ user_profile: 风险偏好/收入
    ↓
格式化为文本摘要（200字以内）
    ↓
注入到 Prompt 中
    ↓
LLM 生成个性化回答
```

**缓存优化**：
- 查询级缓存：LRU 20条（exact match）
- 分析级缓存：按交易哈希缓存
- **效果**：API调用减少60%，响应时间降至0.1秒

**组件3: XAI Explainer**

**决策流程**：
```
Step 1: 规则引擎过滤产品
    ├─ 风险等级匹配：product.risk <= user.max_risk

    └─ 流动性要求：product.liquidity in ["T+0", "T+1"]
    ↓
Step 2: 生成决策日志（JSON）
    {
      "applied_rules": [...],
      "rejected_products": [...]
    }
    ↓
Step 3: LLM转换为自然语言
    Prompt: "用通俗语言解释决策逻辑..."
    ↓
Step 4: 展示给用户
    "为什么推荐这个组合？
     1. 您的风险偏好是...
     2. ..."
```

**监管友好性**：
- 决策规则可追溯（JSON日志）
- 符合银保监会"适当性管理"要求

3. 用户授权访问：每次访问需验证
4. 合规认证：ISO 27001 + 等保三级

**挑战3: 推荐模型的冷启动**

- 新用户无历史数据，推荐质量差

4. 主动学习：用户每次反馈都优化模型

**挑战4: 竞品模仿**

**问题**：
- 技术门槛不高，大厂可能快速复制
